What are the different access specifiers in Java?

Access specifiers are
  1. public, - these class/methods/variables can be accessed by any class or method.
  2. private - class, methods, or variables defined as private can be accessed within the class only.
  3. protected - these can be accessed by the classes of the same package, or by the sub-class of this class in other packages also.
  4. default - Default are accessible by classes within the package only.
  5. There is no default specifier, if no specifier is given, the class will have default access specifier. 
       class A{
       private int a;
       public boolean bl;
       float c;
       protected double d;
       public void print(){
        ....
       }
       protected int geta(){
        ....
      }
   }

What is a constructor?

Constructor is a special method in a class which is used to initialize the object. It is invoked when an object is created.
Note :
  1. Name of the constructor must be same as class name.
  2. > And constructor must not have an explicit return type.

Can a constructor be final? Is the constructor inherited by sub-classes?

No. The constructor of a class can't be final. And constructor is not inherited.

What are the differences between local variables and instance variables?

Local variables are defined in the method. Scope of the local variables is inside the method itself. An instance variable is defined inside the class and outside any methods. Scope of instance variable is throughout the class.

What is a static block in Java?

Java static block or static initializer block is the group of statements that gets executed when the class is loaded Java ClassLoader.
Static block is used to initialize static variables of the class.
A class can have multiple static blocks which are executed in the same order they are defined in the class. 
 
e.g.
   class A{
       static int m;
        int n;
        static{
          m = 1;
       }
       A(int num){
           n = num;
      }
   }

Here the static { m = 1;} is the static initializer block. Unlike the constructor, this block gets executed only once. It initiailizes the static variable m

What is constructor overloading in Java?

Defining multiple constructors for a class is called constructor overloading.
e.g. 
 
public class Test
{
     Test(int n){
         ...
    }
    Test(){
        ...
    }
   public static void main(String arg[]){
      Test t1 = new Test(10);
      Test t2 = new Test();
      ....
  
}
}

Here Test class has two constructors Test(int n), Test(). t1 calls first constructor with one parameter and t2 calls second constructor.

Write a program to read names and times taken by n runners and find the fastest runner. 

 
class Runner{
     String name;
     int runTime;
     Runner(String n, int t){
         name = n;
         runTime = t;
     }
     int getTime(){
         return runTime;
     }
     String getName(){
         return name;
     }
}
class Demo{
     public static void main(String args[]){
            int n;
            System.out.println(""Number of runners "");
            Scanner scanner = new Scanner(System.in);
            n = scanner.nextInt();
            Runner arr[] = new Runner[n];
            for(int i=0;i<n;i++)
            {
                System.out.println(""Name and time taken"");
                String name = scanner.next();
                int time = scanner.nextInt();
                arr[i] = new Runner(name,time);
            }
            int fastest = 0;
            for(int i=1;i<n;i++){
                 int time = arr[i].getTime();
                 if(time<arr[fastest].getTime()) 
                        fastest = i;
            }
            System.out.println(""the fastest runner is ""+arr[fastest].getName());      }
}

We are using a class Runner with 2 fields- name and time taken. Next we create an array of Runner objects.

Find the smallest element in the array. And then find the corresponding Name.

We could even use a comparator or comparable interfaces.
